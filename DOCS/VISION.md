# 1. Présentation

SAMTECH CRM est une application de gestion de la relation client conçue pour les entrepreneurs, indépendants, commerciaux, TPE et PME qui utilisent principalement WhatsApp pour communiquer avec leurs prospects et leurs clients.

Le produit part d'un constat simple : de nombreuses entreprises acquièrent leurs prospects par WhatsApp, mais ne disposent pas d'un système structuré pour enregistrer ces contacts, suivre leurs besoins, organiser les relances et mesurer leur transformation en clients.

Les conversations s'accumulent dans WhatsApp sans constituer une véritable base commerciale exploitable. Des prospects sont oubliés, des relances ne sont pas effectuées et les entreprises disposent de peu d'informations sur les produits réellement demandés ou sur leurs performances commerciales.

SAMTECH CRM vise à résoudre ce problème.

---

# 2. Vision

La vision de SAMTECH CRM est de devenir un outil de gestion commerciale simple, accessible et efficace permettant aux petites entreprises et aux professionnels de transformer leurs contacts et conversations WhatsApp en une véritable base de données commerciale structurée.

SAMTECH CRM doit permettre à l'utilisateur de suivre l'ensemble du cycle commercial :

**Prospect → Qualification → Relance → Conversion → Client → Vente → Facture → Paiement → Fidélisation**

Le produit ne cherche pas à remplacer WhatsApp.

WhatsApp reste le canal de communication.

SAMTECH CRM devient le système qui organise, structure et exploite commercialement les relations créées à travers WhatsApp.

---

# 3. Problème identifié

De nombreux entrepreneurs et petites entreprises utilisent WhatsApp comme principal canal commercial.

Cependant, WhatsApp n'est pas un CRM.

Il permet de communiquer, mais ne permet pas facilement de :

- constituer une base structurée de prospects ;
- classer les prospects par localité ;
- identifier les produits ou services qui les intéressent ;
- connaître les produits les plus demandés ;
- organiser systématiquement les relances ;
- identifier les prospects transformés en clients ;
- calculer le taux de conversion ;
- segmenter une base commerciale ;
- organiser des campagnes ciblées ;
- suivre les achats des clients ;
- facturer les clients ;
- suivre les paiements ;
- analyser les performances commerciales.

Ces informations restent souvent dispersées entre WhatsApp, les contacts du téléphone, des carnets, des fichiers Excel ou la mémoire du responsable commercial.

SAMTECH CRM centralise ces informations dans un environnement unique.

---

# 4. Proposition de valeur

SAMTECH CRM permet à une entreprise de transformer ses contacts commerciaux en une base de données exploitable et orientée vers la conversion.

Le produit doit permettre de répondre simplement à des questions essentielles :

- Qui sont mes prospects ?
- Où se trouvent-ils ?
- Quels produits demandent-ils ?
- Quel est mon produit le plus demandé ?
- Quels prospects dois-je relancer aujourd'hui ?
- Quels prospects sont devenus clients ?
- Quel est mon taux de conversion ?
- Quels clients ont acheté quels produits ?
- Quel chiffre d'affaires ai-je réalisé ?
- Quelles factures restent impayées ?
- Quels groupes de prospects puis-je cibler avec une campagne ?

# 6. Positionnement

SAMTECH CRM se positionne comme un CRM :

**Simple**

L'utilisateur doit pouvoir comprendre l'application rapidement sans formation complexe.

**Mobile-first**

L'expérience utilisateur est conçue en priorité pour le smartphone.

**PWA**

L'application est accessible depuis un navigateur moderne et peut être installée comme une application sur les appareils compatibles.

**Offline-first**

Les fonctions essentielles doivent rester accessibles sans connexion Internet.

**Orienté WhatsApp**

WhatsApp constitue le principal canal de communication commerciale de la V1.

**Accessible**

Le produit doit être adapté aux petites structures qui n'ont pas besoin de CRM complexes.

**Générique**

SAMTECH CRM doit pouvoir être utilisé dans différents secteurs d'activité.

---

# 7. Philosophie produit

Chaque fonctionnalité proposée pour SAMTECH CRM doit respecter cinq principes fondamentaux.

## 7.1 Simplicité

Une fonctionnalité complexe qui peut être simplifiée doit être simplifiée.

## 7.2 Rapidité

Les actions fréquentes doivent nécessiter le moins d'étapes possible.

## 7.3 Utilité

Une fonctionnalité doit répondre à un besoin commercial concret.

## 7.4 Mobilité

Toutes les fonctions essentielles doivent être facilement utilisables sur smartphone.

## 7.5 Résilience

L'utilisateur doit pouvoir continuer à travailler même lorsque sa connexion Internet est absente ou instable.

# 10. Modèle de données de la V1

SAMTECH CRM V1 adopte une approche offline-first.

Les données métier sont stockées localement dans le navigateur à l'aide d'IndexedDB.

Une couche d'accès aux données telle que Dexie.js pourra être utilisée pour simplifier et fiabiliser les opérations.

Les données locales comprennent notamment :

- prospects ;
- clients ;
- produits et services ;
- catégories ;
- localités ;
- intérêts produits ;
- relances ;
- modèles de messages ;
- campagnes ;
- factures ;
- lignes de factures ;
- paiements ;
- notes ;
- paramètres.

Le modèle de données détaillé sera défini dans `DATABASE.md`.

# 12. WhatsApp dans la V1

SAMTECH CRM n'a pas vocation à remplacer WhatsApp.

Dans la V1, l'application doit permettre notamment de :

- préparer un message personnalisé ;
- utiliser des modèles de messages ;
- intégrer des variables telles que le nom du prospect ou le produit demandé ;
- ouvrir la conversation WhatsApp correspondante ;
- faciliter les relances individuelles ;
- organiser des campagnes ciblées assistées.

L'envoi automatisé à grande échelle via l'API officielle WhatsApp Business ne fait pas partie du périmètre initial de la V1.

---

# 13. Fonctionnement des campagnes

Une campagne doit permettre de sélectionner des destinataires selon différents critères.

Exemples :

- localité ;
- produit demandé ;
- statut du prospect ;
- niveau d'intérêt ;
- tags ;
- période d'enregistrement ;
- statut client/prospect ;
- historique d'achat.

L'objectif est de permettre des campagnes plus pertinentes qu'un envoi identique à l'ensemble de la base.

---

# 14. Facturation

Lorsqu'un prospect effectue un achat, il peut être converti en client.

Le système doit permettre de :

- créer une facture ;
- ajouter un ou plusieurs produits ou services ;
- définir les quantités ;
- appliquer les prix ;
- calculer les montants ;
- générer une facture PDF ;
- partager la facture ;
- enregistrer les paiements ;
- suivre les montants restant à payer.
